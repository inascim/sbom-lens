Add two new exports to vulnerabilityAPI.js. Keep all existing exports intact.

**1. parseCVSSSeverity(scoreString) — private helper:**
- Accepts a CVSS vector string or numeric string from OSV's `severity[].score`
- Try to parse numeric base score: use `database_specific` first, then split vector on "/" and look for the last numeric segment
- Return "CRITICAL" (>=9.0), "HIGH" (>=7.0), "MEDIUM" (>=4.0), "LOW" (fallback)
- If scoreString is falsy, return "LOW"

**2. fetchVulnerabilitiesForSBOM(components) — main export:**
- `components` is an array of SBOM component objects with a `purl` field (e.g., `sbom.components`)
- Step 1: Extract PURLs — `components.map(c => c.purl).filter(Boolean)`
- If no PURLs, return []
- Step 2: POST to `https://api.osv.dev/v1/querybatch` with body `{ queries: purls.map(purl => ({ package: { purl } })) }`
- If response.ok is false, return []
- Response shape: `{ results: [{ vulns: [{ id, modified }] }] }` — results array matches purls order
- Build `purlVulnPairs: Array<{ purl, osvId }>` by flattening results
- If no pairs, return []
- Step 3: Collect unique osvIds. Fetch full detail via `GET https://api.osv.dev/v1/vulns/{osvId}` for each unique ID using `Promise.allSettled`. Build `osvIdToDetail` map from fulfilled results.
- Step 4: Assemble records — for each `{ purl, osvId }` pair:
  - Lookup detail from `osvIdToDetail`; skip if missing
  - `cveId` = first alias starting with "CVE-" from `detail.aliases`, fallback to `osvId`
  - `severityEntry` = first entry in `detail.severity` where `type === "CVSS_V3"` or `type === "CVSS_V4"`
  - `severity` = `parseCVSSSeverity(severityEntry?.score)`
  - Return `{ purl, cveId, severity, summary: detail.summary || "", publishedDate: detail.published || new Date().toISOString(), source: "osv", status: "NOT_STARTED", patchAvailable: false }`
- Return assembled records array

**3. Update fetchFromNVD — upgrade from deprecated v1.0 to v2.0:**
- Change URL from `https://services.nvd.nist.gov/rest/json/cves/1.0?keyword=...` to `https://services.nvd.nist.gov/rest/json/cves/2.0?keywordSearch=...`
- Change header from `"api-key"` to `"apiKey"` (NVD v2 header name)
- Update response parsing: v2 shape is `{ vulnerabilities: [{ cve: { id, descriptions, metrics } }] }`
  - `cveId` = `item.cve.id`
  - `severity` = `item.cve.metrics?.cvssMetricV31?.[0]?.cvssData?.baseSeverity || item.cve.metrics?.cvssMetricV30?.[0]?.cvssData?.baseSeverity || "UNKNOWN"`
  - `publishedDate` = `item.cve.published`
  - `description` = `item.cve.descriptions?.find(d => d.lang === "en")?.value || ""`
  - `source` = `"nvd"`
- Keep the `!apiKey` early-return guard (NVD fallback requires key — if no key, skip silently)

**Create src/hooks/useVulnerabilityFetcher.js:**

```javascript
import { useCallback } from "react";
import { useVulnerabilities } from "hooks/useVulnerabilities";
import { fetchVulnerabilitiesForSBOM } from "utils/vulnerabilityAPI";

export const useVulnerabilityFetcher = () => {
  const { vulnerabilities, addVulnerability } = useVulnerabilities();

  const fetchVulnsForSBOM = useCallback(
    async (components) => {
      try {
        const records = await fetchVulnerabilitiesForSBOM(components);

        // Dedup against existing store: skip if cveId+purl already present
        // (manual entries and previously fetched results must not be overwritten)
        const existingKeys = new Set(
          vulnerabilities.map((v) => `${v.cveId}||${v.purl}`)
        );

        records.forEach((rec) => {
          const key = `${rec.cveId}||${rec.purl}`;
          if (!existingKeys.has(key)) {
            addVulnerability(rec);
          }
        });
      } catch (err) {
        // Fire-and-forget: log but do not surface to user
        console.warn("Vulnerability fetch failed silently:", err);
      }
    },
    [vulnerabilities, addVulnerability]
  );

  return { fetchVulnsForSBOM };
};
```
